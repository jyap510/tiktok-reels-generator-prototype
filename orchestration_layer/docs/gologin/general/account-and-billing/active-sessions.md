# Active Sessions & Lock Session

> Understand concurrent cloud session limits and the Lock Session feature

## What is Lock Session?

Lock Session prevents other users from opening the same browser profile while you are actively working in it. When enabled, the profile becomes locked to your session and cannot be launched by teammates until you close it.

This is especially recommended for **LinkedIn** and other session-sensitive platforms where simultaneous access from multiple locations can trigger account bans or security checks.

## How to enable Lock Session

* Click the **lock icon** next to the profile in the profile list.
* Alternatively, open the profile's settings and enable **Lock Session** from there.

Once locked, the profile will show a lock indicator. Other team members will see that the profile is in use and will not be able to launch it until you close your session.

## Cloud launch concurrent session limits

Each plan has a limit on how many browser profiles you can run simultaneously in the cloud (Run in Cloud / web app):

| Plan         | Concurrent cloud sessions |
| ------------ | ------------------------- |
| Professional | 1                         |
| Business     | 2                         |
| Enterprise   | 3                         |
| Custom       | Contact sales             |

<Note>
  These limits apply to **cloud (web) launches only**. Local desktop launches are limited by your hardware, not by your plan.
</Note>

## "Run limit exceeded" error

If you see a **"Run limit exceeded"** error, it means you have reached your plan's concurrent cloud session limit.

To resolve this:

1. **Close an active cloud session** — stop one of the profiles currently running in the cloud.
2. **Upgrade your plan** — switch to a higher plan for more concurrent cloud sessions.
